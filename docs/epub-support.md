# EPUB 支持范围

本项目的 EPUB 路径与 TXT/SRT 文本路径相互独立。导入时保留原始 EPUB 文件，
阅读器按书内 package、manifest、spine 与原生目录构建富内容章节；下载译文时在
临时副本中写入译文，不覆盖 IndexedDB 中保存的原文件。

## 已支持

- EPUB 3 Navigation Document 与 EPUB 2 NCX，包括嵌套目录、文档片段、非线性
  目录目标、重复 spine 条目和无目录时的 spine 回退。
- XHTML 与 EPUB 2 `text/html` 内容文档；扩展名、空格、百分号编码和相对路径均
  通过包内规范路径解析。
- 标题、段落、列表、表格、引用、Ruby、图片、内联 SVG、MathML、音频、视频、
  本地 CSS 与字体。出版物样式在隔离的 Shadow DOM 内生效，不会污染应用界面。
- reflowable 与 pre-paginated 内容、横排与竖排 writing mode、电脑分页和手机/
  电脑连续滚动。
- 同文档与跨文档锚点；HTTP(S) 与 `mailto:` 外部链接以受保护的新标签打开。
- manifest fallback 链、缺失可选资源、非规范 XHTML 的兼容读取。兼容处理会在
  导入后提示，并作为结构化诊断保存在本地书籍元数据中。
- 原文、译文、中日、日中四种阅读模式。翻译仅替换已映射的语义文本块；图片、
  超链接、结构、样式与其他出版资源继续保留，翻译服务输出不会作为 HTML 执行。

## 有意禁用或不支持

- DRM 或未知加密算法保护的正文不能读取，导入时会显示具体受保护资源路径。
- EPUB 脚本、表单、iframe、object、embed 和事件处理器不会执行。
- 远程图片、样式、字体或媒体不会由书籍自动请求；书内普通外部链接仍可由用户
  主动打开。
- IDPF/Adobe 字体混淆不会阻止正文阅读，但当前不会解混淆字体，而是回退到阅读器
  字体并给出诊断。
- Media Overlay 的同步朗读、JavaScript 交互书和厂商专有 DRM 不在支持范围内。

## 故障恢复

缺失图片、字体、样式或其他非正文资源时，阅读器尽量显示其余内容。首选 spine
资源不可读时会沿 manifest `fallback` 链选择 XHTML/HTML；没有任何可读正文时
导入失败，而不会创建一本文字为空的书。目录损坏时按 spine 生成保守目录。

当前应用数据库版本为 6。该版本采用新的富 EPUB 章节投影并按项目策略直接重置
旧应用数据，不执行旧字段迁移。升级前如需保留本地书籍或译文，请先下载；升级后
重新导入原文件。
